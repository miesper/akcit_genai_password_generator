# Backlog Mínimo – Gerador de Senhas CLI

## Release 1: Core
- [ ] **RF01** – Permitir definir o tamanho da senha
  - Critério de aceite: Usuário informa o tamanho e a senha gerada possui exatamente o número de caracteres solicitado.
- [ ] **RF02** – Permitir inclusão/exclusão de letras maiúsculas
  - Critério de aceite: Usuário pode optar por incluir ou não letras maiúsculas na senha.
- [ ] **RF03** – Permitir inclusão/exclusão de letras minúsculas
  - Critério de aceite: Usuário pode optar por incluir ou não letras minúsculas na senha.
- [ ] **RF04** – Permitir inclusão/exclusão de números
  - Critério de aceite: Usuário pode optar por incluir ou não números na senha.
- [ ] **RF05** – Permitir inclusão/exclusão de caracteres especiais
  - Critério de aceite: Usuário pode optar por incluir ou não caracteres especiais na senha.
- [ ] **RF06** – Gerar senha aleatória conforme critérios definidos
  - Critério de aceite: Senha gerada respeita todos os critérios informados pelo usuário.

## Release 2: Qualidade
- [ ] **RT01** – Utilizar gerador de números aleatórios seguro (`secrets`)
  - Critério de aceite: Código utiliza apenas métodos seguros para geração de senhas.
- [ ] **RT02** – Interface CLI robusta a erros de entrada
  - Critério de aceite: Mensagens de erro claras e tratamento de entradas inválidas.
- [ ] **RT03** – Disponibilizar ajuda via `--help`
  - Critério de aceite: Usuário pode acessar documentação dos comandos pela CLI.
- [ ] **RT04** – Código modular, limpo e documentado
  - Critério de aceite: Código segue boas práticas de organização e possui comentários explicativos.

## Release 3: Entrega Final
- [ ] **RT05** – Compatibilidade multiplataforma (Windows, Linux, macOS)
  - Critério de aceite: Aplicação testada e funcional nos principais sistemas operacionais.
- [ ] **RT06** – Não armazenar senhas após exibição
  - Critério de aceite: Senhas não são salvas em disco ou memória após exibição ao usuário.

---

*IDs: RF = Requisito Funcional | RT = Requisito Técnico*
